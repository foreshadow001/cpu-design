# 实验概述

## 1. 实验目的

1. 理解单周期CPU的结构及工作原理
2. 熟悉miniRV或miniLA指令集
3. 掌握单周期CPU的设计与实现方法

### 完整的miniRV指令集分类

miniRV指令集共包含 **44条** 指令，按功能分为7大类。根据教学安排，指令分为A组（浅蓝，18条）、B组（浅橙，18条）以及Demo已实现（绿色，8条），合计44条。

| 类别 | 指令数 | 指令列表 |

| 逻辑运算指令 | 6 | ori, xor, xori, or, and, andi |
| 移位运算指令 | 6 | slli, sll, srl, srli, sra, srai |
| 加载及存储指令 | 8 | lw, lb, lbu, lh, lhu, sw, sb, sh |

### 单周期CPU设计的工程化方法（5步法）

本实验推荐采用以下5个步骤，系统化地完成单周期CPU的设计与实现。

#### 第1步：分析指令 → 构建指令级数据通路

对每一条指令进行逐条分析，为后续设计奠定基础：
- **解析指令格式**：识别指令的 opcode、funct3、funct7、rs1、rs2、rd、imm 等字段的位置和位宽
- **确定操作数来源**：来源可能是寄存器（rs1/rs2），也可能是立即数字段（imm）中的立即数
- **确定结果去向**：结果可能写入寄存器（rd），也可能写入数据存储器
- **确定运算类型**：ALU需要执行何种运算（加减、逻辑、移位、比较等）
- **构建单条数据通路**：为每条指令画出从取指 → 译码 → 执行 → 访存 → 写回的完整数据流路径

#### 第2步：综合数据通路（合并 + 处理多输入）

将所有指令的指令级数据通路进行合并，形成统一的完整数据通路：
- **合并通路**：将各条指令的数据通路叠加在一起，共享相同的功能单元（如ALU、RF）
- **处理多输入冲突**：当某个功能单元的输入存在多个可能的来源时，引入 **MUX（多路选择器）** 或拓宽数据位宽来解决
- **确定MUX选择信号**：为每个MUX定义选择信号，确定每条指令在各MUX上对应的选择值

#### 第3步：构造控制逻辑（提取控制信号 + 填写取值表）

根据综合后的数据通路提取控制逻辑：
- **提取控制信号列表**：如 RegWrite（寄存器写使能）、ALUSrc（ALU第二操作数来源选择）、MemWrite（存储器写使能）、MemtoReg（写回数据来源选择）、Branch（分支使能）、Jump（跳转使能）等
- **填写控制信号取值表**：对每条指令，确定各控制信号在对应指令下的具体取值（0/1或具体值）
- **形成控制器真值表**：以 opcode + funct3 + funct7 的编码组合作为输入，各控制信号作为输出

#### 第4步：提取功能部件（确定接口 + HDL实现）

从数据通路中确定所需的功能部件及其接口：
- **确定部件清单**：PC、NPC、RF、SEXT、ALU、MREQ、MEXT 共7个基本功能部件
- **定义接口信号**：确定每个模块的输入输出端口（端口名、位宽、方向）
- **Verilog HDL实现**：为每个部件创建独立的 `.v` 源文件，编写硬件描述语言代码

#### 第5步：连接和组装

按照综合后的数据通路图将各部件组装为完整的CPU：
- **模块实例化**：在顶层模块（cpu_core）中实例化各功能部件
- **信号连接**：按照数据通路图，使用wire信号连接各模块之间的数据和控制通路
- **控制器接入**：将控制器生成的控制信号分配到各个功能模块的控制端
- **仿真验证**：运行 Basic Trace 测试序列进行功能验证，逐条指令调试

### 7个基本功能部件概览

单周期CPU由以下7个基本功能部件组成：

| 部件 | 全称 | 功能描述 | 所属单元 |
|------|------|----------|----------|
| **PC** | Program Counter | 程序计数器，存储当前正在执行的指令地址 | 取指单元 |
| **NPC** | Next PC | 下一条指令地址计算单元，支持PC+4顺序执行、分支跳转和 jal/jalr 跳转 | 取指单元 |
| **RF** | Register File | 寄存器堆（寄存器文件），提供rs1、rs2的读取和rd的写入 | 执行单元 |
| **SEXT** | Sign Extender | 立即数扩展单元，将指令中编码的立即数字段按I/S/B/U/J型格式扩展为32位 | 译码单元 |
| **ALU** | Arithmetic Logic Unit | 算术逻辑单元，执行所有算术运算、逻辑运算、移位运算和比较运算 | 执行单元 |
| **MREQ** | Memory Request | 存储访问请求单元，负责数据存储器的读写地址和控制信号生成 | 访存单元 |
| **MEXT** | Memory Extension | 存储扩展单元，对load指令从存储器读出的数据进行符号扩展或零扩展 | 访存单元 |
