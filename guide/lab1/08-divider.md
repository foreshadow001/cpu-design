# 除法器设计

> **除法器实现**
>
> 除法器的接口信号及时序与乘法器相同，此处不再赘述。
>
> 点击下载并解压`div_test.zip`。先在`div_test`工程中实现除法器。通过仿真测试后，再把除法器模块集成到单周期CPU工程当中。

此处介绍原码除法器的设计。原码除法器的被除数、除数、商和余数均采用原码表示。

整数的原码除法将符号和数值分开运算。

令 [x]原 = x0 x1 x2 ... xn，[y]原 = y0 y1 y2 ... yn，则有：

```
[x/y]原 = (x0 XOR y0) . (x1 x2 ... xn) / (y1 y2 ... yn)
```

其中，x0、y0 分别为 x、y 的符号；
x1x2...xn 为 x 的绝对值，记作 x*；
y1y2...yn 为 y 的绝对值，记作 y*。

由上式可知，在原码除法运算中，商的值由被除数和除数的绝对值相除求得，即 x* / y*；商的符号由被除数、除数的符号位经过异或运算求得，即 x0 XOR y0。余数的符号与被除数相同。

## 1. 恢复余数法

假设被除数、除数的**数据位宽**均为 n，则基于恢复余数法的原码除法步骤如下：

- **Step1**：令被除数（余数）减去 y*，即加上 [-y*]补
- **Step2**：若得到的余数为负数，表明不够减，故商上 0，并加上 y* 以恢复余数；
  若得到的余数为正数，表明够减，故商上 1
- **Step3**：若左移次数小于 n，令被除数（余数）左移 1 位，回到 Step1；
  否则结束算法，此时的被除数即为余数

> **示例**
>
> 设被除数 x = +1101B，除数 y = -0110B；x 的绝对值 x* = 01101B，y 的绝对值 y* = 00110B，y* 的补码 [-y*]补 = 11010B，则有：

>
> 注意，在本例子中，x 和 y 的数据位宽均为4位（最高位的符号位不算入内），故左移次数应是4。
> 通过观察容易发现，由于被除数的绝对值最多只有4位，故被除数（余数）首次加上 [-y*]补 后，所得结果必定为负，必定需要恢复余数，即第一轮循环的有效操作仅仅是将被除数（余数）左移了一次。由此，可将恢复余数法进行简化，省去第一轮循环。

## 2. 加减交替法

假设被除数、除数的**数据位宽**均为 n，则基于加减交替法的原码除法步骤如下：

- **Step1**：令被除数减去 y*，即加上 [-y*]补
- **Step2**：若得到的余数为负数，表明不够减，故商上 0，令被除数左移 1 位，再加上 [y*]补；
  若得到的余数为正数，表明够减，故商上 1，令被除数左移 1 位，再加上 [-y*]补
- **Step3**：若左移次数已达 n 次，结束，此时的被除数即为余数；否则回到 Step2

> **示例**
>
> 设被除数 x = +1101B，除数 y = -0110B；x 的绝对值 x* = 01101B，y 的绝对值 y* = 00110B，y* 的补码 [-y*]补 = 11010B，则有：
>
> [图8-2 加减交替法示例]
>
> 同样地，x 和 y 的数据位宽均为4位，故原本需左移4次。
> 通过观察容易发现，由于被除数的绝对值最多只有4位，故首次循环时，[-y*]补 总是与被除数（余数）高位的 4'b0000 相加，然后随着被除数（余数）左移1位，接着再加上 [y*]补。这一系列操作可以简化为将 [-y*]补 直接与左移1位的被除数相加，原理如下：
>
> {[-y*]补, x*} << 1 + {[y*]补, 4'b0000} = 2^(4+1) * [-y*]补 + 2^1 * x* + 2^4 * [y*]补
>                                       = 2^4 * (2 * [-y*]补 + [y*]补) + 2^1 * x*
>                                       = 2^4 * [-y*]补 + 2^1 * x*
>                                       = {[-y*]补, 4'b0000} + (x* << 1)

## 3. 除法器参考架构

分析原理可知，原码除法包含求补、加法、左移、异或操作，在运算过程中还需判断被除数（余数）的符号从而选择不同的操作，并在循环一定次数后结束运算。

求补、加法、左移、异或操作很容易通过组合逻辑实现；符号判断可通过特定的控制逻辑实现；而循环次数的控制则可通过计数器实现。

此处给出原码除法器的参考实现架构，如图8-3所示。

[图8-3 原码除法器参考架构]

在图8-3中，深蓝色箭头表示数据，深橙色箭头则表示控制信号。

控制逻辑接受2个输入：一个是被除数（余数）的符号位，另一个则是计数器的计时结束信号。控制逻辑产生3个控制信号：一个控制被除数（余数）需要与 [y*]补 还是与 [-y*]补 相加，即控制图8-3中的多路选择器；另外一个控制被除数（余数）的左移；还有一个控制上商。

复位时，各寄存器均被清零，计数器被复位。运算时，控制逻辑通过判断被除数（余数）最高位的符号位，控制上商以及加法器的数据源选择；加法器将被除数（余数）的高8位与 [y*]补 或 [-y*]补 相加，结果更新到被除数（余数）；此外，控制逻辑接收计数器的计时信号，控制被除数（余数）的移位操作，并在移位6次后结束除法运算。
