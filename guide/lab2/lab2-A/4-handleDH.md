## 1. 数据冒险的检测

  要想解决流水线处理器中的数据冒险问题，首先需要在指令流中检测出数据冒险。

  下面以图4-1为例介绍RAW冒险三种情形的检测方法。

### 1.1 RAW情形A检测

  相邻指令发生RAW冒险 —— 图4-1中，第2条指令在译码阶段访问的寄存器与第1条指令在执行阶段将要写回的寄存器冲突。

  流水线CPU中，与情形A有关的数据通路如图4-2所示。

图4-2 RAW情形A有关的流水线数据通路

  分析图4-2所示的数据通路，不难发现RAW情形A的判断方法/检测逻辑如下：

( REG**ID/EX**.RD == ID.RS1 ) || ( REG**ID/EX**.RD == ID.RS2 )

  其中，REGID/EX表示译码阶段和执行阶段之间的流水线寄存器，下同；REGID/EX.RD表示ID/EX流水线寄存器的RD字段，观察图4-2可知该字段暂存的是从ID级传递而来的目标寄存器的寄存器号。

  显然，对于图4-1中的情形A，有REGID/EX.RD == ID.RS1 == `5'd19`。

### 1.2 RAW情形B检测

图4-3 RAW情形B有关的流水线数据通路

  分析图4-3所示的数据通路，不难发现RAW情形B的判断方法/检测逻辑如下：

( REG**EX/MEM**.RD == ID.RS1 ) || ( REG**EX/MEM**.RD == ID.RS2 )

  显然，对于图4-1中的情形B，有REGEX/MEM.RD == ID.RS1 == `5'd19`。

### 1.3 RAW情形C检测

  间隔2条指令发生RAW冒险 —— 图4-1中，第4条指令在译码阶段访问的寄存器与第1条指令在写回阶段将要写回的寄存器冲突。

## 2. 数据冒险解决方法

### 2.1 流水线暂停

  流水线发生数据冒险时，最朴素的解决方法就是暂停后续的指令，等待前面的指令执行完成，再重新让后续的指令执行。

举栗说明 ![🌰](https://cdn.jsdelivr.net/gh/jdecked/twemoji@15.0.3/assets/svg/1f330.svg)

  对于如图4-1所示的RAW情形A，可让第二条指令暂停3个时钟周期，等待第一条指令写回后，再继续执行，如图4-7所示。

图4-7 流水线暂停

  流水线暂停是功能最强大的冒险解决方法，因此它将流水线中并行进行的操作串行化。暂停法除了可以解决数据冒险，还可用于实现访存指令和乘除法指令。

暂停法实现访存指令、乘除法指令的思路

  访存指令需要通过总线读写主存数据，需要多个时钟周期才能完成。乘除法指令需要通过硬件乘除法器完成运算，而硬件乘除法也需要多个时钟周期才能完成。对于32bit的“IF-ID-EX-MEM-WB”的五级流水CPU，假设采用两位编码的Booth算法实现乘法器，乘法操作至少也需要16个时钟周期才能完成。因此，在不改变流水线深度的前提下，需要借助流水线暂停来延长执行阶段，才能实现乘除法指令。

  使用暂停法实现访存指令的思路是：在EX阶段或MEM阶段通过相应的控制信号（比如`ram_rop`、`ram_wop`等）判断指令是否是访存指令；如果是，则通过MREQ模块产生只有效一个时钟周期的访存请求信号（即`daccess_ren`或`daccess_wen`），同时暂停流水线的IF阶段和ID阶段，直到访存完成（即`daccess_valid`或`daccess_wresp`有效），再令流水线恢复执行。

  使用暂停法实现乘除法指令的思路是：在EX阶段通过`alu_op`判断指令是否是乘除法指令；如果是，则产生只有效一个时钟周期的乘除法启动信号`start`，同时暂停流水线的IF阶段和ID阶段，直到乘除法器输出的`busy`信号从高电平变成低电平，此时乘除法运算完成，再令流水线恢复执行。

  要使流水线的执行暂停下来，只需暂停相关寄存器的更新即可，即：

  （1）保持取指地址（即PC寄存器的值）不变；  

  （2）保持各个流水线寄存器（IF/ID、ID/EX、EX/MEM、MEM/WB）不变；

  暂停寄存器更新的方法是：将寄存器的输入修改为寄存器自身的输出。

### 2.2 数据前递

  数据前递又称数据前推、数据旁路、数据转发，指的是将某个流水线阶段的中间结果向前传递到前面的阶段，且被前递的数据直接参与前面阶段中的操作。

举个栗子 ![🌰](https://cdn.jsdelivr.net/gh/jdecked/twemoji@15.0.3/assets/svg/1f330.svg)

  对于图4-1所示的RAW冒险，可采用数据前递方法解决 —— 将执行阶段、访存阶段或写回阶段的中间结果前递给译码阶段，并让前递而来的数据参与到指令译码时的源操作数选择，如图4-9所示。

图4-9 数据前递示意图

  图4-9中，从EXE指向ID、从MEM指向ID以及从WB指向ID的三条紫色箭头，分别对应图4-1中RAW冒险的三种情形。第一条指令在执行阶段即可产生计算结果（x19寄存器的新值），此时通过执行阶段到译码阶段的数据前递，可让第二条指令在译码时获得x19的新值，从而在不需暂停流水线的情况下解决了第一、二条指令之间的RAW冒险。

### 2.3 乱序执行

  流水线发生数据冒险时，除了暂停和前递，还可以让后续的无相关关系的指令先执行，使得冒险问题自然而然地解决。

  实现乱序执行的基本思路是，检测指令流中是否存在冒险；若存在冒险，则让后续的无相关指令先执行。乱序执行既可通过硬件动态实现，也可以通过编译器或汇编器静态实现。

  典型的乱序执行算法包括记分牌算法和Tomasulo算法等，其硬件实现方法较为复杂，此处不作详细介绍。感兴趣的同学们可自行查阅相关资料。

  对乱序执行感兴趣的同学，建议先尝试通过汇编器静态实现。
