# 实验步骤（详细清单）

## Step 1：下载模板和工程

- [ ] 从课程网站首页的**课程材料下载**链接进入下载页面
- [ ] 下载**数据通路表模板**（Excel/表格文件）
- [ ] 下载**控制信号取值表模板**（Excel/表格文件）
- [ ] 根据**指令集选择**（miniRV 或 miniLA）和**上课实验室编号**，下载对应的 **Vivado demo工程压缩包**
- [ ] 解压Vivado demo工程，确认目录结构完整
- [ ] 用 Vivado 打开 demo 工程，检查工程能否正常加载

> 图12-1 根据实际情况，选择其中一个demo工程

## Step 2：运行仿真，理解demo

- [ ] 在 Vivado 中运行 demo 工程的**功能仿真（Behavioral Simulation）**
- [ ] 打开**波形窗口（Wave Window）**，观察各信号的时序变化
- [ ] 结合**数据通路表**和**控制信号表示例**，对照波形逐条理解 Demo 已实现的 8 条指令（绿色标注）的执行过程
- [ ] 重点关注：PC变化、指令取指、寄存器读写、ALU运算、存储器访问、写回操作
- [ ] 查看 `cpu_core.v` 顶层模块的代码结构，理解各模块的实例化和连接关系

## Step 3：完成数据通路表和控制信号取值表

- [ ] **小组成员分工**：各自认领 A组（浅蓝，18条）或 B组（浅橙，18条）指令
- [ ] 逐条分析各自负责的指令，对照指令格式规范：
  - [ ] 确定指令的 opcode、funct3、funct7 编码
  - [ ] 确定 rs1、rs2、rd 的位置和作用
  - [ ] 确定立即数的编码格式（I型/S型/B型/U型/J型）
- [ ] 为每条指令填充**数据通路表**：
  - [ ] 指令的执行阶段划分（IF → ID → EX → MEM → WB）
  - [ ] 各阶段的数据来源和数据去向
  - [ ] 各MUX的选择信号取值
- [ ] 为每条指令填充**控制信号取值表**：
  - [ ] RegWrite、ALUSrc、MemWrite、MemRead、MemtoReg、Branch、Jump 等
- [ ] 与组员**交叉校对**各自完成的结果，确保理解一致

## Step 4：综合并绘制完整数据通路图

- [ ] **合并数据通路**：将所有指令的指令级数据通路叠加合并
- [ ] **处理多输入问题**：
  - [ ] 对存在多个输入来源的端口添加 **MUX（多路选择器）**
  - [ ] 无法用MUX解决的，考虑**拓宽数据位宽**
- [ ] **标注控制信号**：在每个MUX旁标注对应的控制信号名称
- [ ] **绘制完整数据通路图**（可使用Visio、Draw.io等工具，也可手绘）：
  - [ ] 包含所有7个基本功能部件（PC、NPC、RF、SEXT、ALU、MREQ、MEXT）
  - [ ] 包含所有数据总线和控制总线
  - [ ] 包含指令存储器（IMEM）和数据存储器（DMEM）
  - [ ] 标注所有MUX和主要控制信号

## Step 5：实现功能部件

为每个功能部件创建独立的 `.v` 源文件，使用 Verilog HDL 实现其内部逻辑：

- [ ] **PC**（Program Counter）：实现程序计数器，每个时钟上升沿更新为NPC的值
- [ ] **NPC**（Next PC）：实现下一条指令地址计算，支持 PC+4、分支跳转、jal/jalr 跳转
- [ ] **RF**（Register File）：实现32个32位寄存器的读写操作（同步写、异步读）
- [ ] **SEXT**（Sign Extender）：实现I/S/B/U/J型立即数的扩展逻辑
- [ ] **ALU**（Arithmetic Logic Unit）：实现所有算术、逻辑、移位、比较运算功能
- [ ] **MREQ**（Memory Request）：实现数据存储器读写控制
- [ ] **MEXT**（Memory Extension）：实现load数据的符号扩展/零扩展（lb/lbu/lh/lhu/lw）

> **提示**：每个部件应独立验证其功能正确性，可编写简单的testbench进行单元测试。

## Step 6：实现控制器

- [ ] 根据**控制信号取值表**，提取控制逻辑
- [ ] 使用 **行为描述法（case语句）** 编写控制器模块：
  - [ ] 以 opcode 作为 main decoder 的输入
  - [ ] 以 opcode + funct3 + funct7 作为整体输入组合
- [ ] 输出所有控制信号：RegWrite、ALUSrc、MemWrite、MemtoReg、Branch、Jump、ALUOp 等
- [ ] 实现 **ALU控制器（ALU Control）** 子模块，根据 ALUOp 和 funct3/funct7 生成 ALU 操作码

## Step 7：组装和测试

- [ ] **模块连接**：在 `cpu_core.v` 中实例化所有功能部件，按数据通路图完成信号连接
- [ ] **控制器接入**：将控制器输出的控制信号分配到各功能模块的控制端
- [ ] **单组测试**：
  - [ ] 先通过 A组指令的 **Basic Trace** 测试
  - [ ] 再通过 B组指令的 **Basic Trace** 测试
- [ ] **整合测试**：
  - [ ] 与组员讨论协作，合并A组和B组的实现
  - [ ] 整合后的完整CPU通过 **全部Basic Trace** 测试
- [ ] 如果测试失败，使用Trace比对工具定位错误，返回Step 5/6进行修正
